# CLI reference

`worker observe RUN_ID` reads bounded durable redacted events with `--after-cursor`,
`--limit` (1‚Äì100), and `--timeout-ms` (0‚Äì30000). JSON returns one checkpoint;
`--jsonl` emits events then a checkpoint. Interrupt output separates queued, repeated,
worker-observed, protocol-acknowledged, and terminal phases.
Every observation poll is read-only: it never recovers, reconciles, dispatches, or
otherwise mutates durable run, action, or worker state.

The installed executable is `skein`. Run `skein COMMAND --help` for the exact help
generated by the current binary.

## Output conventions

```console
skein project list                 # human-readable hierarchy
skein --format json project list   # stable machine-readable object
skein project list --format json   # global option may appear later
skein worker watch 42 --jsonl      # event stream
```

`--format human|json` is global and defaults to `human`. Older command-local
`--json` flags remain available. Streaming control and worker operations use
newline-delimited JSON through `--jsonl`.

Long foreground refreshes emit deterministic stage changes to stderr only when
stderr is an interactive terminal and output is human-readable. JSON stdout remains
one document; redirected/non-TTY commands and MCP stdio emit no progress text.

Private prompts are read from standard input by `control codex`, `worker start`,
`worker resume`, `worker steer`, `match`, and `conduct`. Command-line arguments and
process listings therefore do not contain prompt text.

## State and discovery

### `doctor`

Reports executable version, resolved config/data/database paths, database existence,
and readable schema version. It opens no writable registry and performs no migration.

### `init`

Creates private state or applies forward-only migrations. It does not register a
project, index a repository, or contact Codex.

### `freshness`

Reads existing durable timestamps without scanning files, contacting Codex, or
mutating state. Every source reports item count, newest and oldest observation,
newest age and maximum age, and `empty`, `missing`, `fresh`, or `stale`. Git and
project documents report missing coverage when a registered project has no
observation. Staleness uses the oldest covered row, so one recent item cannot hide an
older observation. The default stale boundary is 24 hours and is strict: an age equal
to the boundary is fresh. `mayBeStale` is true for `missing` or `stale` overall state.

Refresh status and freshness answer different questions: `Updated` means indexed
content changed, while `Unchanged` means an authoritative check found the same
content. Both successful observations advance `refreshedAt`; failed or deferred
checks do not.

### `update`

`update [VERSION]` is CLI-only and operates only on an unchanged, receipt-owned
binary release installation. Without `VERSION` it resolves the approved preview
channel. `--check` verifies release metadata and reports availability without
mutation; `--json` returns structured status. Same-version reinstall requires
`--force`. Downgrades are refused unless `--allow-downgrade` is supplied after schema
compatibility review. The compiled binary version, not the mutable receipt version,
is used for comparisons and reporting; disagreement is refused. Source/manual
installs are directed to the source installer.

### `project add|show|refresh|list`

- `add PATH [--name NAME]` canonicalizes one existing directory.
- `show PATH` renders its project card and latest observations.
- `refresh PATH` or `refresh --all` refreshes Git metadata. `--working-tree` checks
  tracked changes; `--force` ignores the stored fingerprint.
- `list` uses stable display order.

### `scan-root add|list|remove`

`add` persists discovery authority and immediately discovers repositories. Recursion
is explicit; depth defaults to 16 and cannot exceed 64. `remove` accepts a stored path
even if it is offline and does not remove discovered projects.

### `index`

Runs the coordinated discovery, Git, identity-document, enabled-context, and Codex
session refresh described in [indexing and search](indexing-and-search.md).

### `search`

Returns ranked projects and project-document hits. `--include-session-text` only
permits names/previews previously imported with `--include-text`. `--deep-context`
explicitly adds enabled private context hits; `recall` reports consulted sources,
authorization, freshness, limits, truncation, and escalation guidance.

## Private context

### `context status`

Shows independent generated-memory and raw-session settings, document counts, source
status, and last refresh information.

`context refresh` and global `index` reports identify raw-session preparation as
`full`, `incremental`, `unchanged`, `fallback_full`, `disabled`, or `deferred`, with
`bytesRead`, `recordsConsidered`, `filesReused`, `filesIncremental`, `filesFull`,
`fallbackFiles`, and `filesDeleted` accounting.

### `context memories enable|disable`

Changes generated memory-summary recall. A refresh applies the new source set.

### `context sessions enable|disable`

Changes raw user/assistant session-message recall. Approved-root and canonical-cwd
checks still apply.

### `context refresh`

Atomically rebuilds enabled sources. `--codex-home` overrides Codex discovery for
this call; `--max-files` defaults to the authoritative 10,000-file bound.

### `context search`

Searches only opted-in private context and returns bounded snippets.

### `conduct`

Automatic dispatch requires one unique high-confidence route. Ranked candidates expose
stable project/session selectors. `--project-id ID` selects one ranked project and
optional `--session-id THREAD` selects its ranked resumable session; both are
revalidated transactionally against the unchanged stdin prompt.

## Codex session metadata

### `import codex preview`

Reads one bounded app-server page and writes no Session Skein state. Text is omitted
unless `--include-text`; JSONL repair is omitted unless `--repair-source-index`.

### `session sync codex`

Transactionally imports metadata. One page defaults to 50 threads. `--all-pages`
follows opaque cursors within the page/thread caps. `--since-days` filters source
updates. Text import and source-index repair are separate explicit flags.

### `session search`

Searches currently enabled private Codex-session projections and exact one-rollout
generated-memory summaries. It returns bounded ranked hits with source provenance,
exact thread IDs, source dates/path, optional cwd/project, match mode, and a ready
`codex resume THREAD_ID` command. All normalized terms are required first;
any-term fallback occurs only when strict search has no hits. Search-only opens state
read-only. `--refresh` explicitly refreshes enabled recall sources first.

### `session list|show|bind|unbind`

Lists or resolves opaque source IDs. `bind` requires an existing registered project;
`unbind` preserves the session while deliberately removing its project relationship.
The default source is `codex`.

## Control and workers

### `control codex`

Runs one foreground turn against an exact registered project or cataloged thread.
`--full-access` is required. `--include-content` only changes live display; Session
Skein still does not store prompt or model output.

### `control list|show|mark-stale`

Reads redaction-safe audit records. `mark-stale --force` quarantines records only
after checking controller liveness; the flag acknowledges the residual race.

### `worker start|resume`

Creates one reconnectable per-run process. The private prompt comes from stdin.
`--follow` stays attached; `--jsonl` follows with structured events.

### `worker list|status|watch|read`

Reads redacted durable or live state. `read` asks Codex for exact source identity and
status without resuming the turn.

### `worker steer|interrupt|reconcile|stop`

- `steer` sends stdin text to the exact active turn and accepts an idempotency UUID.
- `interrupt` targets the exact active turn.
- `reconcile` reads a recovery-required source turn and durably records evidence; it
  never replays or takes over work.
- `stop` refuses an active run and removes only an idle or terminal worker.

## Matching, conductor, and summaries

### `match`

Reads a query from stdin, ranks locally, and never dispatches. Limit defaults to 5.

### `conduct`

Reads one prompt from stdin and requires `--full-access`. A UUID makes a lost retry a
status lookup. `--follow` streams redacted human events; `--jsonl` streams structured
route, worker, event, and terminal records.

### `summary projects|project|day`

Renders deterministic prose from existing metadata. It launches no Codex process,
Git command, LLM, or repository scan.

## Presentation and integration

### `tui`

Starts the standalone keyboard-first interface. See [TUI](tui.md).

### `mcp`

Starts stdio MCP and refuses an interactive terminal. Without `--allow-control`, four
worker-control tools are omitted. Server startup initializes/migrates private state so
a first tool can return setup guidance. See [MCP reference](mcp-reference.md).

## Exit and error behavior

Success exits zero. Invalid arguments, unavailable required paths, failed protocol
preconditions, policy refusal, or partial command-level failure exits nonzero and
writes the error to stderr. Individual `index` source failures are represented in a
successful aggregate report when safe progress was preserved.
